CWE Top 25 Most Dangerous Software Weaknesses (2023)

The CWE Top 25 is a demonstrative list of the most common and impactful software weaknesses. This list is calculated by analyzing public vulnerability data in the U.S. National Vulnerability Database (NVD) for the previous two calendar years.

Top 10 Most Dangerous Software Weaknesses

  1. CWE-787: Out-of-bounds Write

    The product writes data past the end, or before the beginning, of the intended buffer. This typically occurs when pointer arithmetic is used to reference or modify data in a buffer and the programmer miscalculates how much data can be safely written.

    Score: 65.93

    Related Weaknesses:

    • CWE-120: Buffer Copy without Checking Size of Input
    • CWE-131: Incorrect Calculation of Buffer Size
    • CWE-134: Use of Externally-Controlled Format String

  2. CWE-79: Cross-site Scripting

    The product does not neutralize or incorrectly neutralizes user-controllable input before it is placed in output that is used as a web page that is served to other users. This weakness is prevalent in applications that generate web pages dynamically.

    Score: 45.54

    Related Weaknesses:

    • CWE-80: Improper Neutralization of Script-Related HTML Tags in a Web Page
    • CWE-83: Improper Neutralization of Script in Attributes in a Web Page
    • CWE-87: Improper Neutralization of Alternate XSS Syntax

  3. CWE-89: SQL Injection

    The product constructs all or part of an SQL command using externally-influenced input from an upstream component, but it does not neutralize or incorrectly neutralizes special elements that could modify the intended SQL command when it is sent to a downstream component.

    Score: 34.27

    Related Weaknesses:

    • CWE-564: SQL Injection: Hibernate
    • CWE-652: Improper Neutralization of Data within XQuery Expressions
    • CWE-943: Improper Neutralization of Special Elements in Data Query Logic

  4. CWE-416: Use After Free

    Referencing memory after it has been freed can cause a program to crash, use unexpected values, or execute code. The use of previously-freed memory can have any number of adverse consequences, ranging from the corruption of valid data to the execution of arbitrary code.

    Score: 16.71

    Related Weaknesses:

    • CWE-415: Double Free
    • CWE-459: Incomplete Cleanup
    • CWE-672: Operation on a Resource after Expiration or Release

  5. CWE-78: OS Command Injection

    The product constructs all or part of an OS command using externally-influenced input from an upstream component, but it does not neutralize or incorrectly neutralizes special elements that could modify the intended OS command when it is sent to a downstream component.

    Score: 15.65

    Related Weaknesses:

    • CWE-88: Improper Neutralization of Argument Delimiters in a Command
    • CWE-93: Improper Neutralization of CRLF Sequences
    • CWE-94: Improper Control of Generation of Code

  6. CWE-20: Improper Input Validation

    The product receives input or data, but it does not validate or incorrectly validates that the input has the properties that are required to process the data safely and correctly. Input validation is a frequently-used technique for checking potentially dangerous inputs.

    Score: 15.50

    Related Weaknesses:

    • CWE-129: Improper Validation of Array Index
    • CWE-1284: Improper Validation of Specified Quantity in Input
    • CWE-190: Integer Overflow or Wraparound

  7. CWE-125: Out-of-bounds Read

    The product reads data past the end, or before the beginning, of the intended buffer. This typically occurs when the pointer arithmetic is used to reference or modify data beyond the buffer boundaries, or when array indices exceed the valid range.

    Score: 14.60

    Related Weaknesses:

    • CWE-126: Buffer Over-read
    • CWE-127: Buffer Under-read
    • CWE-824: Access of Uninitialized Pointer

  8. CWE-22: Path Traversal

    The product uses external input to construct a pathname that is intended to identify a file or directory that is located underneath a restricted parent directory, but the product does not properly neutralize special elements within the pathname that can cause the pathname to resolve to a location that is outside of the restricted directory.

    Score: 14.11

    Related Weaknesses:

    • CWE-23: Relative Path Traversal
    • CWE-36: Absolute Path Traversal
    • CWE-73: External Control of File Name or Path

  9. CWE-352: Cross-Site Request Forgery

    The web application does not, or can not, sufficiently verify whether a well-formed, valid, consistent request was intentionally provided by the user who submitted the request. When a web server is designed to receive a request from a client without any mechanism for verifying that it was intentionally sent, then it might be possible for an attacker to trick a client into making an unintentional request to the web server.

    Score: 11.73

    Related Weaknesses:

    • CWE-346: Origin Validation Error
    • CWE-613: Insufficient Session Expiration
    • CWE-829: Inclusion of Functionality from Untrusted Control Sphere

  10. CWE-434: Unrestricted Upload of File with Dangerous Type

    The product allows the attacker to upload or transfer files of dangerous types that can be automatically processed within the product's environment. This can allow the attacker to upload, install, and execute arbitrary code or other files.

    Score: 8.45

    Related Weaknesses:

    • CWE-184: Incomplete List of Disallowed Inputs
    • CWE-436: Interpretation Conflict
    • CWE-646: Reliance on File Name or Extension for File Type Identification

About the CWE Top 25

The CWE Top 25 list is based on real-world vulnerabilities reported to the National Vulnerability Database (NVD). The scoring formula combines the frequency of occurrence with the average CVSS score to identify the most dangerous weaknesses.

Methodology

The list is calculated by analyzing CVE records from the previous two years, mapping them to CWE identifiers, and calculating a composite score based on prevalence and average CVSS impact scores.

Common Weakness Enumeration (CWE) is a community-developed list of software and hardware weakness types sponsored by the Cybersecurity and Infrastructure Security Agency (CISA) and maintained by the MITRE Corporation.
